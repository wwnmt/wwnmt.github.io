端口扫描器

  端口扫描器可以侦察出目标主机开放的端口以及相应的服务程序版本，这有利于攻击者寻找漏洞，进行下一步的渗透攻击。在这里以python为基础，编写了一个可以基于多线程，多进程等的tcp协议端口扫描程序，可以探测端口开放情况以及相应服务。未来还可以在此基础上，加上更复杂的扫描类型，如各种标志位组合的tcp扫描，udp扫描，系统扫描等，就像nmap一样。但是还是推荐使用nmap这个强大的工具，自己的工具除了学习以外，在更复杂的环境下可能不适用。

用上多线程

  我们的目标不仅仅局限于此，当对一个网段和所有端口进行大规模扫描时，这个扫描器的效率并不能满足需要，因此让使用多线程，对每个端口开启一个线程去扫描，伴随来的除了效率的提高。但是操作系统对于线程数有一个上限（在KALI AMD64 下是1024个线程），端口数远远大于1024。除此之外在打印扫描结果时，会发生死锁等问题，线程会相互插队，打印结果惨不忍睹。针对每一个host开启一个线程去扫描是可取的，针对每一个端口去开启一个线程是不可取的。这里只给出针对少量端口进行多线程扫描的例子:

非传统方法-异步协程

  上面的是传统的方法，建立一个线程池，默认情况下socket操作阻塞，线程调用connect、recv、send等操作时，会暂停，进行I/O操作时也会暂停。针对很多主机和端口，需要很多线程，然而线程的代价不低，如果在主机或端口分配完之前，就用完了线程，那是很头疼的事。而且对与每个主机开启一个线程，在这个线程上进行的端口扫描也不是并发的，仅仅针对了主机的并发扫描，效率依旧一般。
  异步I/O框架可以使用非阻塞套接字在单个线程上实现并发任务。
